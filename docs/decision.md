# 결정사항

실제 Search를 수행하는 객체와 시뮬레이션을 수행하는 객체는 시행에서 몇 번 호출될일 없기에 동적 다형성 사용.

내부에서 실제로 연산을 수행하는 객체는 매우 많이 호출되기에 정적 다형성 사용.

메모리 파편화 방지와 캐시 적중률을 높이기 위해 커스텀 메모리 할당자 사용.

거의 대부분 같은 크기의 데이터를 할당 및 해제하기에 메모리 할당자는 풀(pool) 할당자로 결정.

최대한 캐시 효율성을 높이려 State에 있는 효과와 대미지를 State 객체에 ptr과 각각 효과와 데미지 배열의 오프셋과 길이를 저장하는 것으로 구현.

이펙트를 각 캐릭터 스킬에 저장하고 State에서는 이펙트 유무 키를 이용하여 데미지와 이펙트 유무를 저장으로 변경.
